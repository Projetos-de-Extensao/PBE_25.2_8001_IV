# Mapa de Jornadas de Usuário — Documento inicial

Objetivo: mapear quais páginas devem existir e quais são prioritárias para cada tipo de usuário.

## Personas / Tipos de usuário (resumo)
- Aluno — busca entender o projeto, protótipos e requisitos.
- Professor / Orientador — busca diagramas, critérios, relatórios e visão arquitetural.


---

## Jornada: Aluno
- Objetivo: Buscar informaçoes sobre monitorias e se candidatar.
- Pontos de entrada: `index.md` (Home), `Iniciacao/index.md`, `Elaboracao/Exemplo Biblioteca Digital.md`.
- Tarefas principais:
  1. Ler a visão geral (Home)
  2. Ver protótipos (Iniciacao/prototipo_baixa_fidelidade.md, Elaboracao/prototipo_alta_fidelidade.md)
  3. Consultar FAQ/recursos (jornadas/jornada_usuario.md)
- Páginas prioritárias (3):
  - `index.md`
  - `Iniciacao/prototipo_baixa_fidelidade.md`
  - `Elaboracao/Requisitos.md`

---

## Jornada: Professor 
- Objetivo: avaliar o aluno para possivel monitoria.
- Pontos de entrada: `diagramas-test/README.md`, `Elaboracao/diagrama_de_classes.md`, `Elaboracao/diagrama_de_sequencia.md`.
- Tarefas principais:
  1. Ver diagramas de alto nível
  2. Ler requisitos e casos de uso
  3. Acessar material de avaliação (notas/entregas)
- Páginas prioritárias (3):
  - `diagramas-test/README.md`
  - `Elaboracao/Requisitos.md`
  - `Elaboracao/casos_de_uso.md`
